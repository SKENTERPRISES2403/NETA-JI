# NETA JI Production Roadmap

## Stage 1: Pitch PWA

- Public demo link.
- Mobile-friendly first session.
- Safe fictional party setup.
- Shareable result poster.
- Offline PWA behavior.

Status: in progress and deployable.

## Stage 2: Content Polish

- Better cartoon sprites for neta, volunteers, voters, flag carriers, dhol team, camera crew, and rival supporters.
- More readable state map and selected-state flow.
- Stronger victory ceremony.
- More sound and haptic feedback.
- More safe fictional events.

## Stage 3: APK Review Build

- Android wrapper.
- App icon and splash.
- Real-device QA.
- Store-safe privacy and safety copy.
- Internal test APK.

## Stage 4: Moderated Public Sharing

Use a backend only when public user-generated content is stored or shared.

Required systems:

- Server-side name and slogan moderation.
- Blocklist updates without app release.
- Report button for posters or names.
- Review queue for flagged content.
- Rate limits for poster generation and sharing.
- Delete/takedown tools.
- Audit logs for moderation actions.

## Stage 5: Multiplayer

Multiplayer should wait until the single-player arcade loop is fun and stable.

Likely backend needs:

- Matchmaking or room codes.
- Authoritative game state.
- Anti-spam controls.
- Moderated party names.
- Replay/result validation.
